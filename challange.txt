Seu desafio é construir o backend API REST de uma aplicação de entregas, você poderá escolher detalhes da aplicação:  nome, tipo de item da entrega, e etc.

Para esse projeto, é obrigatório o uso da stack: Typescript, NestJS, PrismaORM e MySQL. 

Sua aplicação deverá ser capaz de:

Obrigatório: 
[OK] -> Criar a entrega de um item. --- CREATE
[NO] -> Procurar pela entrega de um item específico. --- GET ONLY
[NO] -> Procurar entregas por destinatário ou por remetente. --- GET MANY
[OK] -> Atualizar o estado como entregue.  --- UPDATE
[OK] -> Cancelar uma entrega.  --- DELETE
[NO] -> Consumir uma outra API REST já existente para buscar dados. --- GET EVEN MORE DATA
## Sugestão exemplo: receber apenas o CEP e o número dos endereços e 
buscar os detalhes desse endereço em outra API, 
como bairro, rua, e geolocalização.  ---  https://viacep.com.br

Diferenciais (não obrigatórios): 
[NO] -> Solução do problema.
[NO] -> Fluxo de Autenticação
[NO] -> Documentação (README ou Swagger)
[NO] -> Boas práticas (clean code, clean architecture, design patterns e etc)
[NO] -> POO
Caso concorde com o teste, defina um prazo para entregá-lo e converse comigo sobre.

Lembre-se que a vaga é imediata e urgente, então você definir o prazo é muito importante. Outras pessoas também estão fazendo o teste.
